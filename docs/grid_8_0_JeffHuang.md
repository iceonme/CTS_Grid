2. 核心原理
2.1 网格计算原理
2.1.1 数据采集
时间窗口：过去6小时（360分钟）
采样频率：5个时间段，每段约72分钟
数据点：每个时间段取1个高点 + 1个低点
2.1.2 抗插针处理（5取3）
plain
复制
原始5个高点：[H1, H2, H3, H4, H5] 按大小排序
去掉：最高1个（H5）+ 最低1个（H1）
保留：H2, H3, H4 取平均 → 网格顶部

原始5个低点：[L1, L2, L3, L4, L5] 按大小排序  
去掉：最低1个（L1）+ 最高1个（L5）
保留：L2, L3, L4 取平均 → 网格底部
2.1.3 波动率计算
plain
复制
波动率 = (网格顶部 - 网格底部) / 网格底部

> 1.2% → 7层模式（高波动）
≤ 1.2% → 5层模式（低波动）
2.2 层数结构设计
2.2.1 5层模式（低波动）
表格
层号	类型	价格区间	区域	操作规则
-2	虚拟	底部-2h ~ 底部-h	买入区	RSI < 25
-1	虚拟	底部-h ~ 底部	买入区	RSI < 25
0 (5)	实体	底部 ~ 底部+h	买入区	直接买入，无需RSI
1 (4)	实体	底部+h ~ 底部+2h	缓冲层	观望
2 (3)	实体	底部+2h ~ 顶部-2h	缓冲层	观望
3 (2)	实体	顶部-2h ~ 顶部-h	卖出区	直接卖出
4 (1)	实体	顶部-h ~ 顶部	卖出区	直接卖出
+1	虚拟	顶部 ~ 顶部+h	卖出区	RSI > 75
+2	虚拟	顶部+h ~ 顶部+2h	卖出区	RSI > 75
注：h = (顶部-底部)/5
2.2.2 7层模式（高波动）
层高度减小，网格更密集，交易频率提升。

3.3 数据流
plain
复制
1. 获取1分钟K线 → 计算RSI/ATR
2. 每小时更新 → 6小时网格计算 → 确定层数模式
3. 价格触发 → 确定所在层 → 检查周期状态
4. 风控检查 → 熔断/黑天鹅判断
5. 信号生成 → 实体层直接交易 / 虚拟层RSI过滤
6. 订单执行 → 仓位更新 → TG播报
4. 信号系统
4.1 买入信号
表格
条件	实体层（0层）	虚拟层（-1, -2层）
价格位置	触及层下半部	触及层内
RSI要求	无需	< 25（动态调整）
周期检查	该层无持仓	该层无持仓
熔断状态	未触发	未触发
4.2 卖出信号
表格
条件	实体层（3, 4层）	虚拟层（+1, +2层）
价格位置	触及层上半部	触及层内
RSI要求	无需	> 75（动态调整）
周期检查	该层有持仓	该层有持仓
熔断状态	未触发	未触发
4.3 周期管理原则
plain
复制
单次买入 → 该层锁定 → 卖出后解锁 → 可再次买入

禁止行为：
- 同层重复买入（未卖出前）
- 未买入直接卖出
- 熔断期内新开仓
5. 风险管理
5.1 双向熔断机制
5.1.1 触发条件
表格
方向	触发价格	说明
向下熔断	< 虚拟层-2底部	价格跌破网格底部40%
向上熔断	> 虚拟层+2顶部	价格涨破网格顶部40%
5.1.2 熔断后动作
plain
复制
触发 → 立即停止新开仓 → 进入2小时观察期 → 评估：

情况A：观察期内价格回归网格内
      → 熔断解除，恢复原网格运行
      
情况B：观察期结束价格仍在网格外  
      → 以当前资金/持仓为基准，重置6小时网格
5.2 黑天鹅检测（ATR异常）
表格
指标	计算方式	触发条件
当前ATR	过去1分钟真实波幅	-
历史ATR均值	过去6小时ATR均值	-
触发阈值	历史ATR × 3	当前ATR > 阈值
触发后动作：
停止新开仓
每10分钟评估一次，逐步减仓10%
若价格反弹超成本价 → 停止减仓，恢复正常
若持续恶化 → 完成清仓，等待下一周期
5.3 连续熔断保护
plain
复制
24小时内熔断次数 > 3次 → 强制空仓12小时 → 重新评估市场状态

version: "8.0-OPT-FINAL-2025"
codename: "Kimibigclaw"

# ==========================================
# 交易基础配置
# ==========================================
trading:
  symbol: BTC-USDT
  total_capital: 10000          # USDT本金
  max_position_pct: 0.8         # 最大仓位80%
  fee_rate: 0.001               # 手续费率0.1%
  slippage: 0.0005              # 滑点0.05%

# ==========================================
# 网格计算配置
# ==========================================
grid:
  lookback_hours: 6             # 回望周期6小时
  sample_points: 5              # 采样5个高点/低点
  trim_extremes: 2              # 去掉最高最低各1个（5取3）
  rebalance_interval_minutes: 60 # 每小时重新平衡

# ==========================================
# 层数结构配置
# ==========================================
layer:
  base_layers: 5                # 基础层数5层
  virtual_layers: 2             # 上下虚拟层各2层
  volatility_threshold: 0.012   # 波动率阈值1.2%
  # >1.2%切换7层，≤1.2%保持5层

# ==========================================
# RSI信号配置（动态调整）
# ==========================================
rsi:
  period: 14                    # RSI计算周期
  dynamic_adjustment: true      # 开启动态调整
  
  # 默认阈值
  buy_threshold: 25
  sell_threshold: 75
  
  # 动态调整规则
  adjustment_factors:
    high_volatility:            # 波动率>2%
      volatility_min: 0.02
      buy: 20
      sell: 80
    normal:                     # 默认（1.2%-2%）
      buy: 25
      sell: 75
    low_volatility:             # 波动率<1.2%
      volatility_max: 0.012
      buy: 30
      sell: 70

# ==========================================
# 熔断机制配置
# ==========================================
circuit_breaker:
  enabled: true
  trigger_beyond_virtual: true  # 突破2层虚拟层触发
  cooldown_hours: 1.5           # 观察期1.5小时
  consecutive_limit: 3          # 24小时最大熔断3次
  action: 'observe'             # 观察后智能决策
  
  # 智能决策规则
  smart_logic:
    extreme_up: 'partial_exit'   # 暴涨>30%部分减仓
    extreme_down: 'hold'         # 暴跌>30%持仓等待
    normal: 'observe'            # 正常观察

# ==========================================
# 黑天鹅风控配置
# ==========================================
black_swan:
  enabled: true
  atr_lookback_hours: 6
  atr_multiplier: 3             # ATR放大3倍触发
  gradual_exit_interval_minutes: 10
  stop_exit_if_profitable: true
  max_daily_exits: 5            # 日最大减仓5次

# ==========================================
# Telegram Bot配置
# ==========================================
telegram:
  bot_token: "YOUR_BOT_TOKEN"
  chat_id: "YOUR_CHAT_ID"
  alert_threshold: 0.02         # 2%变动播报
  status_interval_minutes: 5    # 每5分钟状态播报
