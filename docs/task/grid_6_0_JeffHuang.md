1. 策略概述
1.1 设计背景
动态网格策略在加密货币市场面临核心矛盾：震荡市需要高频交易捕捉波动，趋势市需要过滤信号避免逆势操作。单一时间框架无法同时满足灵敏度与稳定性要求。
V6.0-MTF 采用多时间框架（Multi-Timeframe, MTF）架构，通过周期错位解决这一矛盾：
短周期（5分钟）：负责精准入场，利用 RSI 捕捉超卖/超买拐点
长周期（15分钟）：负责趋势过滤，利用 MACD 识别多空方向
1.2 核心创新
表格
创新点	说明	解决的问题
周期解耦	入场与趋势判断分离	避免单一周期下的信号冲突
动态网格	基于近期高低点自动计算网格边界	适应不同波动率环境
趋势自适应止盈	根据 MACD 强度调整 RSI 卖出阈值	牛市减少卖飞，熊市及时止损
黑天鹅熔断	ATR 异常放大时自动暂停交易	防范极端行情下的连续亏损
1.3 适用场景
标的：BTC、ETH 等高流动性加密货币现货
市场状态：震荡市（最优）、趋势市（ filtered ）、极端行情（保护）
资金规模：1万-100万美元（需相应调整网格层数）
时间周期：中短线（持仓周期数小时至数周）
2. 核心原理
2.1 多时间框架理论
金融市场存在分形结构，不同时间周期呈现自相似特征。短周期噪声多但拐点灵敏，长周期信号稳但滞后明显。MTF 策略通过信息分层处理实现优势互补：
plain
复制
信息层级：
├──  tick 级别（毫秒）：微观结构，策略忽略
├──  1-5 分钟：短期波动，RSI 捕捉拐点
├──  15-60 分钟：中期趋势，MACD 确认方向
└──  日线以上：长期趋势，策略不直接采用
2.2 双周期协同机制
表格
数据流	周期	指标	职能	更新频率
主数据流	5分钟	RSI、ATR、价格	入场时机、风控、执行	每5分钟
辅数据流	15分钟	MACD	趋势过滤、方向确认	每15分钟
协同逻辑：
每5分钟检查一次交易机会（主周期）
每次检查时读取最新的15分钟 MACD 状态（辅周期）
仅当双周期信号同向时执行交易
周期冲突时（如5分钟超卖但15分钟空头），降级为观望
2.3 动态网格算法
2.3.1 网格初始化
基于近期价格区间自动构建，而非固定价格：
plain
复制
输入：最近6小时（72根5分钟K线）的高低点
计算：
  上沿 = 最高价 × (1 - 缓冲率)    // 预留2%缓冲
  下沿 = 最低价 × (1 + 缓冲率)    // 预留2%缓冲
  区间 = 上沿 - 下沿
  层距 = 区间 / 网格层数
  
生成网格线：[下沿, 下沿+层距, 下沿+2×层距, ..., 上沿]
2.3.2 非对称仓位管理
越接近下沿，买入量越大（金字塔加仓）：
plain
复制
第 n 层买入量 = 总资金 / 网格层数 × (n / 平均层数)
例如5层网格，资金10,000美元：
第1层（最接近下沿）：买入 10,000/5 × (5/3) = $3,333
第3层（中间）：买入 10,000/5 × (3/3) = $2,000
第5层（最接近上沿）：买入 10,000/5 × (1/3) = $667
2.3.3 网格动态调整
当价格突破原网格边界超过5%时，触发网格重置：
表格
突破方向	处理方式	目的
向上突破	上移网格，保留部分仓位跟随趋势	避免完全卖飞
向下突破	下移网格或暂停买入，评估止损	防止持续抄底亏损

5.2 黑天鹅应对机制
定义：在短时间（< 1小时）内，价格出现> 5%的极端波动，或波动率（ATR）异常放大。
检测流程：
plain
复制
1. 计算当前5分钟ATR（14周期）
2. 计算过去6小时（72根）ATR均值
3. 计算比率 = 当前ATR / 历史均值
4. 若比率 > 3.0，触发黑天鹅警报
5. 若比率 > 2.0 且 单根K线幅度 > 2%，预警
应对等级：
表格
等级	ATR比率	K线幅度	措施
预警	2.0-3.0	> 2%	减少仓位50%，TG通知
轻度	3.0-4.0	> 3%	暂停新开仓，移动止盈
中度	4.0-5.0	> 5%	平掉50%仓位，冷却30分钟
重度	> 5.0	> 8%	全仓平仓，人工确认恢复
