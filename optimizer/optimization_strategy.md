# V6.0 策略自动优化策略文档 (Optimization Strategy)

## 1. 优化背景
针对 V6.0 策略中约 23 个核心参数，手动调优无法覆盖所有可能的参数耦合关系。通过自动化搜寻，可以快速定位在历史行情中表现最健壮的参数组合。

## 2. 优化指标 (Objective)
我们不单纯追求最高收益率（防止过拟合），优化目标函数设定为：
`Score = Sharpe_Ratio * (1 - Max_Drawdown) * log(Trade_Count)`
- **夏普率**: 衡量风险调整后的收益。
- **最大回撤惩罚**: 确保系统不会选择极端高波动的参数。
- **交易频率补偿**: 排除那些因参数过严导致全年只交易 1 次的无效组合。

## 3. 搜寻范围 (Search Space)
| 参数名 | 类型 | 范围 | 说明 |
| :--- | :--- | :--- | :--- |
| `rsi_buy_threshold` | int | 20 ~ 35 | RSI 买入超卖线 |
| `rsi_sell_threshold` | int | 65 ~ 80 | RSI 卖出超盈线 |
| `macd_fast` | int | 8 ~ 16 | MACD 快线周期 |
| `macd_slow` | int | 20 ~ 35 | MACD 慢线周期 |
| `atr_blackswan_mult`| float | 2.5 ~ 5.0 | 黑天鹅熔断系数 |
| `grid_layers` | int | 3 ~ 8 | 网格层数 |
| `grid_buffer` | float | 0.01 ~ 0.04 | 网格缓冲区比例 |

## 4. 实验流程
1. **采样阶段**: 使用 2025 年 1-8 月数据作为 **训练集 (In-Sample)**。
2. **搜寻阶段**: 运行 1000 次蒙特卡洛随机搜索或贝叶斯优化。
3. **验证阶段**: 挑选前 5 名结果，在 2025 年 9-12 月数据上作为 **测试集 (Out-of-Sample)** 进行盲测。
4. **归档**: 将最终表现均衡的参数存入 `grid_v60_runtime.json`。

## 5. 工具说明
由于环境限制，暂不依赖外部 Optuna 库，采用内置的多进程并行随机搜索引擎实现。所有中间结果存储在 `optimizer/study_log.csv`。
